%\section*{What is \pharo?}
\section*{Was ist \pharo?}

%\pharo is a modern, open source, fully-featured implementation of the \st programming language and environment. \pharo is derived from \squeak\cite{Inga97a}, a re-implementation of the classic \st-80 system. Whereas \squeak was developed mainly as a platform for developing experimental educational software, \pharo strives to offer a lean, open-source platform for professional software development, and a robust and stable platform for research and development into dynamic languages and environments. \pharo serves as the reference implementation for the Seaside web development framework.
\pharo ist eine moderne, voll ausgestattete Implementierung der \st Programmiersprache  und Entwicklungsumgebung und ist OpenSource. \pharo ist ein Ableger von \squeak\cite{Inga97a}, eine Neuimplementierung des klassischen \st-80 Systems. \squeak wurde hauptsächlich entwickelt als Plattform zur Entwicklung experimenteller und lehrreicher Software. \pharo strebt an, eine schlanke OpenSource Plattform für professionelle Softwareentwicklung zu sein und ebenso eine robuste und stabile Plattform für die Forschung und die Entwicklung von dynmaischen Sprachen und Umgebungen. \pharo dient als Referenz Implementierung für das "Seaside Web Entwicklungs Framework".

%\pharo resolves some licensing issues with \squeak. Unlike previous versions of \squeak, the \pharo core contains only code that has been contributed under the MIT license. The \pharo project started in March 2008 as a fork of \squeak 3.9, and the first 1.0 beta version was released on July 31, 2009.
\pharo löst einige Probleme, die es mit der Lizens von \squeak gab. Anders als die vorherigen Versionen von \squeak, beinhaltet der Kerncode von \pharo nur Kode der unter der MIT Lizens zur verfügung steht. Das \pharo Projekt startete im März 2008 als Zweig von \squeak 3.9. Die Erste 1.0 Beta Version wurde freigegeben am 31. Juli 2009.

%Although \pharo removes many packages from \squeak, it also includes numerous features that are optional in \squeak. For example, true type fonts are bundled into \pharo. \pharo also includes support for true block closures. The user interfaces has been simplified and revised.
Obwohl einige Packete von \squeak aus \pharo entfernt wurden, beinhaltet es eine große Anzahl von Merkmalen, die bei \squeak nur optional waren, so zum Beispiel die Truetype Fonts. Ausserdem bietet \pharo eine echte Unterstüttzung für Block Closures. Das User Interface wurde ebenfalls neu gestaltet und vereinfacht.

%\pharo is highly portable --- even its virtual machine is written entirely in \st, making it easy to debug, analyze, and change. \pharo is the vehicle for a wide range of innovative projects from multimedia applications and educational platforms to commercial web development environments. 
\pahro ist höchst portabel --- sogar die virtuelle Maschine wurde in \st geschrieben, was es einfach macht, diese zu debuggen, zu analysieren und zu ändern. \pharo ist ein Vehikel für einen großen Bereich von innovativen Projekten von multimedia Anwendungen, Lehrreichen Plattformen und kommerziellen Web Entwicklungsumgebungen.

%There is an important aspect behind \pharo: \pharo should not just be a copy of the past but really \emph{reinvent} Smalltalk. Big-bang approaches rarely succeed. \pharo will really favor evolutionary and incremental changes. We want to be able to experiment with important new features or libraries. Evolution means that \pharo accepts mistakes and is not aiming for the next perfect solution in one big step\,---\,even if we would love it. \pharo will favor small incremental changes but a multitude of them. The success of \pharo depends on the contributions of its community.
% The \pharo community will pay attention to your submissions to improve the system.
Es gibt einen wichtigen Aspekt hinter \pharo: \pharo sollte nicht nur eine Kopie aus der Vergangenheit sein sondern ein \emph{Neuerfundenes} Smalltalk. "Big-bang approaches rarely succeed". \pharo begünstigt evolotionäre und inkrementelle Veränderungen. Wir wollen es möglich machen mit neue Merkmalen und Bibliotheken zu experimentieren. Evolution bedeutet \pharo ackzeptiert Fehler und hat somit nicht das Ziel einer perfekten Lösung in einem Schritt \,---\, sogar wenn es uns gefallen würde. \pharo begünstigt kleine inkrementelle Änderungen, aber auch eine große Anzahl von diesen. Der Erfolg von \pharo hängst von den Mitwirkungen seiner Gemeinschaft ab.
